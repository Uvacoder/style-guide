Style Guide

Introduction

Compiling a style guide creates a sense unity and continuity that can serve as a foundation to build friendly and familiar properties across the web. Style guides create foundational design, but it also creates a codebase that can be reused and built upon. This style guide is definitely a living document. Changes will be tracked on Github.

Layout

Row

Rows use flexbox for layout. Widths are automatically proportioned based on the number of divs in the row. Default rows feature a 1.5% gutter between cells. A container is not required to use a row.

 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 

Row with Collapsed Gutter

In some layouts, a gutter may not be desired. Gutter can be removed by adding a utility class.

 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 

Full Width

Rows not inside a container will expand to fit the entire width of their parent element.

 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 

Typography

Typeface

Helvetica Neue Regular is the primary typeface for all text elements with appropriate font and system font fallbacks.

The quick brown fox jumped over the lazy dog.

Handwriting

Rock Salt is a handwriting typface used for text that should call attention to itself. The handwriting typeface should be used sparingly and never for blocks of text. Centering the text is preferred.

Highlights

Heading elements can be highlighted to feature a color background that wraps with the text. Headings with highlights are transformed to uppercase.

Bet your bottom dollar that tomorrow there'll be sun!

Links

Ensuring links are visually represented across the entire color pallette is important. They should have a dotted, 1px bottom border that matches the current color of the text.

The text color of anchor tags found on White, Cloud, or Mist should be colored with Water. On hover, they should change to Steam.

Example Link

Links found on color backgrounds should match their current text color. On hover, the bottom border should change from dotted to solid.

Buttons

Buttons can be used on anchor, input[type="button], input[type="submit"], and button tags.

Basic Button

A basic button features a background color that contrasts its container. On hover, the color darkens and returns to normal when active.

Ghost Button

A ghost button has the same background color as its container. On hover, it fills and the text color changes to the default contrasting color. when active.

Ghost on Color

Ghost on color is utility that enables specifying the text color on hover.

Disabled Button

Disabled buttons are universal in their color and behavior.
